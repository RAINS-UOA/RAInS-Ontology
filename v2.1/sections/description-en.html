The RAInS Ontology: Description back to ToC

The RAInS ontology is an ex-tension of the System Accountability Ontology (SAO) for the AI systemsâ€™ domain by defining a set of concepts required to document the design and implementation stage of such systems. Subclasses of sao:AccountableAction and sao:AccountableResult are defined to provide a minimal set of high-level constructs for describing accountability plans consisting of actions producing design specifications(e.g. a ML model de-sign specification) and human decisions(e.g. approval of a specification by an accountable person). This ontology also defines a number of sao:InformationElement(s) that can be used as members of sao:InformationRealization to describe the actual sao:AccountableResult(s) in the accountability trace. The ontology will be extended in the future to cover additional system life cycle stages such as deployment and operation. See also https://w3id.org/sao. .
